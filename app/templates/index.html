2. Reglas de Disparo Múltiple

El Hub siempre envía un evento de posición base a RC. Las Reglas de Disparo te permiten clonar ese evento y enviarlo con un Código RC distinto según el estado de un sensor (ej: Motor, Puerta, Pánico).

Ejemplo: Podés crear una regla que envíe el código 33 si doorstatus == 1 (Abierta), y otra regla distinta que envíe el código 34 si doorstatus == 0 (Cerrada). Si la regla hace match, el evento se clona y se envía.

4. Diccionario de Metadatos

¿Para qué sirve esto? Básicamente se usa cuando el proveedor nos envía posiciones identificadas con un ID de hardware (como el imei) pero Recurso Confiable nos exige enviar la Patente o Placa del vehículo.

Al configurarlo, el Hub consultará periódicamente una API externa del proveedor para descargar el diccionario de asignaciones y traducirá automáticamente los IMEI a Patentes al vuelo antes de enviarlos a RC.

Solo afecta la visualización. RC siempre recibe UTC 0.
